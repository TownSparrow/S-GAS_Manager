## О проекте
Данный проект является прототипом системы, который применяет алгоритм адаптивного распределения данных с применением семантико-графовой оценки текстовых фрагментов (чанков) при работе с малыми языковыми моделями. Позволяет языковым моделям с поддержкой генерации с дополненной выборкой (RAG) управлять большими контекстами путем адаптивного свопинга чанков между памятью GPU и ОЗУ. Применяет семантическо-графовую оценку ценности каждого из текстовых фрагментов перед каждым запросом для динамического управления информацией на потребительском оборудовании. 

Проект и алгоритм разработаны студентом Леонидом Воробьевым в рамках научно-исследовательской работы магистерской программы "Технологии разработки компьютерных игр" факультета Школа разработки видеоигр Университета ИТМО.

### Научная проблема
Малые языковые модели ограничены фиксированным контекстным окном, что создает критические проблемы при:
- многооборотных диалогах, когда модель теряет ранний контекст и генерирует противоречивые ответы;
- RAG-системах, когда простое накопление чанков приводит к переполнению памяти GPU;
- использовании длинных документах, когда модель теряет информацию в середине контекста.

### Ключевые компоненты алгоритма
Алгоритм включает в себя три ключевых механизма:
1. Семантическое сходство между эмбеддингами запроса и фрагментов
2. Графовая оценка на основе анализа сущностей и их отношений через граф знаний
3. Динамическое перемещение фрагментов между VRAM и RAM на основе полученных оценок

### Ключевые технологии
| Компонент | Технология |
|-----------|-----------|
| Инференс-ядро | vLLM (PagedAttention) |
| Эмбеддинги | Sentence-Transformers |
| Векторное хранилище | ChromaDB (HNSW) |
| Граф знаний | NetworkX + spaCy |
| API | FastAPI + Uvicorn |
| Управление памятью | PyTorch + CUDA |

### Целевые результаты
1. Снижение потребления VRAM на 15-20% при сохранении качества работы системы
2. Повышение качества метрики Recall@K на 5-10% в многооборотных диалогах
3. Обеспечение интерактивной латентности (<200 ms на токен)
4. Обеспечение работы с контекстом, который превышает физический размер видеопамяти минимум в 2 раза

## Быстрый старт

### Требования
- **ОС**: Ubuntu 22.04 LTS (или любой другой поддерживаемый для vLLM дистрибутив Linux)
- **GPU**: NVIDIA GPU с поддержкой CUDA (12GB+ VRAM рекомендуется)
- **CPU**: Минимум 8 ядер
- **RAM**: 16GB+
- **Python**: 3.10+

### Установка системы
Для установки системы достаточно скачать актуальную версию репозитория, перейти в корневой каталог проекта и запустить установочный скрипт:
```bash
chmod +x ./scripts/install_env.sh
./scripts/install_env.sh
```

### Запуск системы
1. Проверить конфигурационный файл и скорректировать по необходимости параметры запуска в `cfg/system_params.json`
2. Открыть в корневой папке проекта терминал и выполнить скрипт запуска:
```bash
chmod +x ./scripts/start_vllm_server.sh
./scripts/start_vllm_server.sh
```
3. Дождаться успешного запуска системы
4. В корневой папке проекта открыть дополнительный терминал, не закрывая предыдущий, и запустить скрипт для запуска API локального сервера:
```bash
source S-GAS_Manager_env/bin/activate
uvicorn run:app --reload --host 0.0.0.0 --port 8080
```
5. После успешного запуска API открыть страницу системы в браузере: **http://localhost:8080**

## Как использовать систему

### Через веб-интерфейс
1. Для создания запроса используйте форму для отправки сообщения. После заполнения нажмите на соответствующую кнопку для отправки.
2. RAG-режим автоматически включается, когда используются документы. Для загрузки документов применяется соответствующая кнопка. Успешная загрузка и обработка документа сопровождается характерным сообщением в чате.
3. При запуске страницы чата создается сессия с уникальным ID. Этот ID необходим для перехода в страницы статистики конкретной сессии.

### Ключевые endpoint'ы для проверки системы
- `/api/session/{session_id}/info` - информация о текущей сессии
- `/api/session/{session_id}/documents` - информация об использованных документах в системе
- `/api/sgas-statistics` - общее состояние алгоритма системы

### Как проверять и собирать логи системы
1. Используйте соответствующие endpoint'ы для просмотра статистики использования системы
2. Общий файл логирования по состоянию системы сохраняется в папку `logs`. Название файла по умолчанию `session_metrics.jsonl`
3. Во время использования системы сообщения о работе системы также выводятся в терминал


### Возможные проблемы и их решение
| Проблема | Решение |
|----------|---------|
| vLLM не запускается | Проверьте поддержку CUDA: `nvidia-smi` |
| API не отвечает | Убедитесь, что vLLM использует правильный порт (8000) |
| Ошибка памяти GPU | Уменьшите потребление видеопамяти с помощью параметра `gpu_memory_utilization` в файле `cfg/system_params.json` |
| Граф не строится | Проверьте, что установлена русская spaCy модель: `python -m spacy download ru_core_news_sm` |

## Архитектура системы
```
S-GAS_MANAGER/
├── run.py                            # Точка входа: DI, маршруты FastAPI, запуск
├── config.py                         # Загрузка конфигурации из JSON
├── requirements.txt                  # Зависимости (основные)
├── requirements_dev.txt              # Зависимости (разработка)
├── app/
│   ├── consts/                       # Константы и шаблоны промптов
│   ├── models/                       # Модели данных (Chunk, Document, Session, API-схемы)
│   ├── interfaces/                   # Интерфейсы сервисов (контракты)
│   ├── services/                     # Бизнес-логика (Embedding, VectorStore, Graph, Scoring, Swap, Chat)
│   │   ├── _processors/             # Процессоры NER и извлечения ключевых слов
│   │   ├── monitoring/              # Мониторинг KV-кеша
│   │   └── testing/                 # Бенчмарк-раннер, метрики, оценщики
│   ├── controllers/                  # Тонкие HTTP-обработчики (Session, Document, Search, Chat, Benchmark)
│   ├── loaders/                      # Загрузчики документов (PDF, Text, DOCX)
│   └── utils/                        # Утилиты (GPU, сериализация, валидация)
├── cfg/
│   └── system_params.json            # Параметры запуска системы
├── static/                           # Веб-клиент (HTML, CSS, JS)
├── scripts/
│   ├── install_env.sh                # Автоматическая установка зависимостей
│   └── start_vllm_server.sh          # Запуск vLLM сервера
├── logs/                             # Логирование метрик сессий
└── data/                             # Хранилище документов и эмбеддингов (создаётся автоматически)
```
